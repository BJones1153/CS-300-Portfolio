What was the problem you were solving in the projects for this course?

The primary problem I was solving was how to organize and retrieve course information efficiently through the analysis of different data structures. In the first project I compared vectors, has tables, and a 
binary search tree to determine which data structure would be best implemented to sort and search course data. In project two, I developed a binary search tree to build a program that could load course 
information from a CSV file, display the course information in alphanumeric order, and display information about the specific course and its prerequisites. 

How did you approach the problem? Consider why data structures are important to understand.

I first approached the problem by initially understanding the different data structures and determining what each one was good for. I first compared vectors, hash tables, and binary search trees to see which
one would work best for sorting and searching the course data. Understanding the different data structures was essential because the one you choose can have a big impact on how fast and efficient your program 
is and executes. After I carefully compared them, I decided a binary search tree was the most optimal choice because it could keep courses in order while still making it easy to traverse the data structure for 
a specific course. 

How did you overcome any roadblocks you encountered while going through the activities or project?

One of the biggest challenges I encountered during this course and the in the projects was debugging my code when things didn’t work the way I expected based on the user requirements. I had problems with the 
directory on my computer when uploading files to Microsoft Visual Studio. I also faced problems when searching for certain courses and getting parts of the binary search tree to function correctly. I was able 
to overcome those problems by breaking the code into smaller pieces and testing one part at a time until I found the defect. It was frustrating at times, but it helped me understand what my program was doing 
instead of just theorizing what I could do to fix it. 

How has your work on this project expanded your approach to designing software and developing programs?

After completing this course, the projects and other assignments taught me to allocate more time planning before I start writing code. Instead of jumping in into programming, I learned to think about the 
various data structures and their role in the current problem I am facing and which one works best. In addition, how to break the program into smaller functions.  I also learned that writing efficient code is 
just as important as getting the program to work correctly. There is a lot of planning involved in these types of projects such as determining which programming language to implement and which data structure is
most efficient based on the problem. Overall, I believe that I have a better understanding of how good planning and choosing the right algorithms can make a program more efficient and easier to develop. 

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

One of the ways my work changed the way I write programs is by making me think more about the organization of the project and readability. Instead of putting everything into one section of code, I started 
breaking my programs into smaller increments in the form of functions that each have a specific job. I also learned more about why it is essential to use clear variable and function names so the code is easier
